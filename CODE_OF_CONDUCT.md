# Code of Conduct — กติกาการอยู่ร่วมกันของทีม

> ฉบับสั้นสำหรับทีมภายใน — กติกาการ*ทำงาน*อยู่ที่ [CONTRIBUTING.md](CONTRIBUTING.md) ไฟล์นี้คือกติกาการ*ปฏิบัติต่อกัน*

## หลัก 5 ข้อ

1. **ถกที่ประเด็น ไม่ใช่ตัวคน** — "query นี้ full scan เพราะ JOIN กรอง status" ✅ · "เขียนแบบนี้ได้ไง" ❌
2. **Review เพื่อให้ของดีขึ้น ไม่ใช่เพื่อชนะ** — ทุก comment ใน PR ต้องมีเหตุผลหรือหลักฐานแนบ (line number, บทเรียนเก่า, ผลวัด) และผู้เขียนตอบทุก comment ก่อน merge
3. **ความเห็นแย้งจบใน PR/Issue** — ตกลงกันไม่ได้ให้เจ้าของ repo ตัดสิน แล้วบันทึกเหตุผลไว้ ไม่ลากยาวข้ามช่องทาง ไม่รื้อหลัง merge โดยไม่มีข้อมูลใหม่
4. **ผิดพลาดแล้วบอกเร็ว ไม่มีการตำหนิย้อนหลัง** — deploy พัง/ข้อมูลเสีย รีบแจ้ง + เดิน rollback ตาม playbook สำคัญกว่าหาคนผิด · บทเรียนแปลงเป็นกับดักใน `CLAUDE.md` §Hard rules หรือ skill `tbt-ds-pitfalls` แทนการจำชื่อคน
5. **เคารพเวลาและ context ของกันและกัน** — งานมีเจ้าของแล้ว (assignee) ไม่ทำซ้อน · จะแตะไฟล์ที่ PR อื่นกำลังแก้ ให้คุยกันก่อน · ข้อมูลลูกค้าอ่อนไหวปฏิบัติตาม [SECURITY.md](SECURITY.md)

## ขอบเขตและการบังคับใช้

ใช้กับทุกพื้นที่ของ repo นี้ (Issue, PR, review, commit message) และช่องทางสื่อสารของทีมที่คุยเรื่องงานนี้ · พฤติกรรมที่ผิดกติกาซ้ำหลังตักเตือน รายงานเจ้าของ repo (@wichtking) พิจารณาเป็นรายกรณี — ตั้งแต่ตักเตือนจนถึงถอนสิทธิ์เข้า repo
